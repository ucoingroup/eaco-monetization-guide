## 一、VPN行业现状与痛点

### 1.1 市场规模

全球VPN市场2024年规模约450亿美元，预计2030年达到**1.07万亿美元**（CAGR ~17%）。增长驱动力：数据主权立法、远程办公常态化、网络审查加剧、隐私意识觉醒。

### 1.2 中心化VPN三大痛点

| 痛点 | 具体表现 |
|------|---------|
| **可审查性** | 中心化VPN运营商受司法管辖，可被强制交出日志、封禁端口、限速特定流量。ExpressVPN、NordVPN等头部玩家均发生过服务器被扣押事件。 |
| **价格高昂** | 主流VPN月费$5-15，年费$60-150。对发展中国家用户（东南亚、拉美、非洲）构成显著经济门槛。 |
| **数据隐私风险** | "无日志"承诺无法验证。2023年SuperVPN数据泄露事件暴露1400万用户信息；多家免费VPN被证实兜售用户浏览数据。 |

### 1.3 市场空白

去中心化VPN（dVPN）仍处早期，现有方案（Orchid、Tachyon、Mysterium）市占率不足2%。核心问题：**节点供给不足、用户体验差、缺乏经济飞轮**。EACO-VPN的切入点正是用$e代币经济解决供给端和需求端的冷启动问题。

## 七、抗审查设计

### 7.1 三层防御体系

**第一层：流量混淆**
- 所有VPN流量封装为标准HTTPS（TLS 1.3）
- 使用域前置（Domain Fronting）：流量伪装为访问CDN上的合法网站
- 流量填充：固定包长+随机填充，消除流量大小特征
- 协议模拟：可模拟HTTP/2、QUIC等常见协议特征

**第二层：多跳路由**
- 默认2跳，可选3-5跳
- 每跳节点在不同司法管辖区
- 路由决策：避开用户所在国和目标国的节点
- 入口节点仅知道用户IP，出口节点仅知道目标IP，无单点全知

**第三层：区块链验证**
- 节点注册信息链上存证，防篡改
- 流量证明链上可验证，防止中间人注入
- 节点信誉评分公开透明，社区可审计
- 去中心化节点发现：无中心服务器可被封禁

### 7.2 应急机制

| 攻击场景 | 应对策略 |
|---------|---------|
| 入口节点IP被封 | 自动切换备用入口 + 混淆协议升级 |
| 出口节点被定位 | 出口IP轮换池（每10分钟换IP） |
| 协议特征被识别 | 混淆层热更新，社区贡献新伪装规则 |
| DNS污染 | DoH/DoT内置，绕过本地DNS |
| 全局断网 | 卫星/网状网备份通道（Phase 4） |

## 八、合规策略

### 8.1 法律框架适配

EACO-VPN作为去中心化协议，不运营、不控制节点，定位为**基础设施协议**而非VPN服务商：

- **协议层免责**：EACO-VPN仅提供开源软件和链上协议，不提供VPN服务本身
- **节点运营者责任**：节点运营者需自行遵守所在地法律，协议层不记录、不审查流量
- **用户协议**：明确告知用户遵守所在国家/地区法律，协议不对用户行为负责

### 8.2 合规措施

| 措施 | 说明 |
|------|------|
| KYC分层 | 基础服务无需KYC；高流量/企业节点需KYC（合作第三方） |
| 黑名单机制 | 出口节点可配置目标黑名单（非法内容），由节点运营者自主决定 |
| 数据最小化 | 零日志架构，链上仅存流量证明哈希，不存任何用户数据 |
| 管辖区隔离 | 节点注册时声明管辖区，路由层自动遵守数据不出境规则（如EU节点流量不出EU） |

### 8.3 牌照与注册

- 注册实体于对加密友好管辖区（如瑞士、新加坡、巴拿马）
- 协议开发实体与节点运营实体分离
- 法律顾问团队持续跟踪各国VPN监管动态
