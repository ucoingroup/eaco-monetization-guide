## 二、Agent World 利润分配：$e vs SOL

### 当前状态分析

| 项目 | 当前机制 | $e Token 改造 |
|------|----------|--------------|
| 结算方式 | 未明确（可能 SOL） | **以 $e 为主结算单位** |
| 利润分配 | 50%再投资 / 50%提现 | 保持比例，但**单位改为 $e** |
| 提现路径 | 钱包 → CEX → 法币 | $e → DEX → SOL/USDT → 法币 |
| Agent 奖励 | 协议未知 | **$e 奖励 = Agent 贡献证明** |

### 推荐的 $e 结算机制

```
Agent World 利润流 (当前 match_1777000323458)

    ┌─────────────────────────────────────┐
    │   Agent World Match 奖金/收益 (SOL)  │
    └──────────────┬──────────────────────┘
                   │
            自动兑换 (DEX)
                   │
                   ▼
    ┌──────────────────────────────────────┐
    │         $e Token Pool                │
    └──────┬──────────────┬───────────────┘
           │              │
    ┌──────▼──────┐ ┌────▼──────────┐
    │ 50% 再投资   │ │ 50% 提现      │
    │ → 质押/回购  │ │ → $e → SOL    │
    │ → 推升价格   │ │ → CEX → 法币  │
    └─────────────┘ └──────────────┘
```

### 关键设计原则

1. **SOL 收入 → $e 购买压力**: Agent World 产出的 SOL/USDT，50%再投资部分通过 DEX 自动买入 $e，制造持续买入压力
2. **提现自由**: 50% 提现部分直接兑现，不受 $e 价格波动影响
3. **$e 价值锚定**: 通过 Agent World 持续产出来支撑 $e 价值，形成飞轮

### 子系统详细设计

#### A. EACO-VPN 带宽激励

| 组件 | 设计 | 技术方案 |
|------|------|---------|
| VPN 协议 | WireGuard + Solana 智能合约 | 轻量、高性能 |
| 节点角色 | Relay + Exit Node | 8 Agent 服务器 + 社区节点 |
| 带宽计量 | IPFS/Orbit 链上记录 | 不可篡改 |
| 激励公式 | `奖励 = min(可用带宽 × 在线时长, 上限)` | 防作弊 |
| Slashing | 节点下线>24h → 扣质押 | 保网络质量 |
| $e 发放 | 每日自动合约执行 | 零人工干预 |
